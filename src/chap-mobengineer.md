# メール管理から考えるデータマネジメント

<div class="flush-right">
モブエンジニア@mob_engineer
</div>

データマネジメントと言われて「とっつきづらそう」「なんか難しそう」といった印象を持たれるかもしれません。実は、皆さんも意識しないところでデータマネジメントをを利用しています。今回は、読者の多くが利用しているであろう「メール」からデータマネジメントを考えてみたいと思います。

## そもそもメール管理って？

職場でもプライベートでもメールを利用されていると思います。特に職場であれば「〇〇社から来たメールは重要だから最重要フォルダに入れておこう」「DMメールは基本読まないから自動的にゴミ箱へ捨てておこう」といろいろ試行錯誤されていると思います。このように、「届いたメールをすぐに取り出せるように保管する」ことをメール管理と呼んでいます。

### メール管理の基本的な流れ

一般的には次の流れで処理を行っていると思います。

1. メールを受信する
2. 「宛先」「タイトル」「本文」を読む
3. メールの内容を類推して「対応可否」を判断する
   1. 対応が必要＋即時対応できる内容であればメールを返して「完了済み」として処理する
   2. 対応が必要だが即時対応できない内容であれば「対応中」フォルダに格納しておく
   3. 対応が不要な内容であれば「対応不要」フォルダに格納しておく
4. 一定期間したらメールデータを整理する

もちろん、フォルダの管理方法によって「会社単位で分ける」「年月日で分ける」といった違いはあると思いますが、メール管理の流れとしては先ほどお示しした内容から大きく変わらないと思います。

### メール管理を自動化させることも

「会社単位」「年月日単位」「差出人ごとに分類する」などの要望が出てきたら手動での振り分け作業は現実的ではありません。Outlookであれば「メールの自動振り分けルール」を利用して「指定した差出人からメールが届いたら任意の場所に格納する」ルールを仕込むことも可能です。もちろん、メール返信ルールを組み込むことでメール対応を半自動化させることも可能になります。

## 自動化≠意識しなくていい

メール管理を自動化させることの最大のメリットは「工数削減」と「メール振り分けに意識を注がなくてよくなる」があります。とは言え、振り分けルールな必要なレベルのフォルダ構成そのものを見直した方がいいのではないかと思います。メールの状態だけで考えれば「未対応」「対応中」「対応済み」の3種類を遷移しているだけにすぎません。メール振り分け機能を活用することも大事ですが、メール振り分けする目的について考え直すことも大事だと考えています。

### データマネジメントも同じ

メール管理も究極的にはデータベースで行っているデータ管理と同じです。大量の情報を「収集」「分類」「加工」していくことで、価値ある情報に昇華させることが出来ます。バックアップ戦略についてもメールの重要度に応じてバックアップ方法を適切に選ぶ必要があります。もちろん、所属組織のルールによって変わってくると思いますので、一概にこの期間保管するのが最適といえるわけではないですが、一般的には「重要事項が含まれているメールは10年」「業務上の情報が含まれているメールは5年」「その他のメールは1年」と言われています。

## おわりに

メール管理からデータマネジメントについて考えてみました。
今後、「生成AIへの利活用」「データの民主化」などのビッグトピックによってデータマネジメントの重要度は高まっていくと思います。本記事を通じて、データマネジメントについて少しでも興味を持っていただければ幸いです。
