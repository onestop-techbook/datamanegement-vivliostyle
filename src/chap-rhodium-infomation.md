# データと情報について


データと情報は、現代では当たり前のように使われる言葉ですが、その違いを厳密に説明しようとすると曖昧になります。本稿では、データと情報の違いを「意味」と「価値」の観点から整理し、なぜその違いを考える必要があるのかを考察します。

## データと情報の定義

まず、データと情報の定義について確認します。アメリカ国立標準技術研究所（NIST）は、データを次のように定義しています。

> "A representation of information, including digital and non-digital formats, from which understandable information is derived. (仮訳：デジタルおよび非デジタル形式を含む情報の表現であり、そこから理解可能な情報が導き出される)" [1]

一方で NIST は、情報について次のように述べています。

> "A meaningful expression of data. (仮訳：データの意味のある表現)" [1] 

> "Any communication or representation of knowledge such as facts, data, or opinions in any medium or form, including textual, numerical, graphic, cartographic, narrative, electronic, or audiovisual forms. (仮訳：テキスト、数値、グラフィック、地図、物語、電子、または視聴覚のあらゆる形態や媒体や形態で行われる事実、データ、意見などの知識の伝達や表現)"[2]

ここから分かるのは、データはあくまで記号の形で保存された素材であり、それを解釈してはじめて情報になるということです。

### 「111」を例に考える

次に、本稿では数値「111」を例に考えていきます。しかし、ここで重要な前提があります。何の文脈もなく「111」とだけ提示されても、それが何を意味するのかは分かりません。暗証番号かもしれませんし、単なる通し番号かもしれません。この「分からなさ」こそが、データと情報を区別する出発点になります。

では、この数値に文脈を一つ与えてみます。「111 mmHg」と単位を付けると、これは血圧の値である可能性が高まります。さらに「血圧 111 mmHg」と対象を示せば、身体に関する情報として理解できるようになります。この時点で、医療の現場では「正常に近い値である」と判断する手がかりを提供できます。つまり、文脈が付与されるごとに、同じ「111」という数値であっても、情報としての価値は段階的に高まっていくのです。

さらに、情報には別の側面として「不確実性の減少」という価値もあります。情報理論の基礎を築いたクロード・シャノンは、情報を「予測できなさの減少量」として捉えました。珍しい値であるほど得られる情報量は大きくなりますが、医療の文脈では少し異なる見方ができます。たとえば血圧が111mmHgという結果は、「高血圧である可能性が低い」という不確実性の除去につながります。つまり、異常が見つからなかったという事実も、不必要な検査や不安を取り除くという意味で価値ある情報なのです。正常範囲であるという情報が得られることで、次に取るべき行動を適切に選択できるようになります。

このことから、情報の価値は一つの尺度だけでは測れないことが分かります。

## 情報表現の違いと情報の質

次に、「情報の表現」の違いについて考えます。情報の価値には「何を表すか」という内容面だけでなく、「どのように表すか」という表現面も影響します。つまり、同じ内容であっても、その表現形式によって再利用性や理解のしやすさが変わり、情報価値も異なるのです。たとえば「太郎の脚が痛い」という文があったとします。人間にとっては理解しやすい情報ですが、機械にとっては意味を正確に把握することが難しい場合があります。

一方で、次のような構造化された表現に変換すると、情報の内容が明確になります。

```Java
class Person {
    String name;
    BodyPart leg;
}

class BodyPart {
    String name;
    String status;
}

Person taro = new Person();
taro.name = "太郎";

BodyPart leg = new BodyPart();
leg.name = "脚";
leg.status = "痛い";

taro.leg = leg;
```
このようにエンティティ（人間、身体部位）と属性（痛み）が明示されていると、機械処理や推論が可能になり、他のデータとの連携も容易になります。情報が再利用される機会が増えるため、情報としての価値が高まると言えます。一方で、こうした構造化表現は人間にとって読みにくく、意味を把握する負担が増えるという側面もあります。ただし、こうした構造化には労力が必要なため、すべての情報をこの形にすべきとは限りません。人間が理解しやすい自然言語と、機械が扱いやすい構造化データには、それぞれ役割があるのです。

## 実用的な視点から見たデータと情報

最後に、データと情報の違いを実用的な観点から整理します。データは保存や管理の対象となる素材であり、そのままでは価値が十分に引き出されないことがあります。一方で情報は、文脈と意味が付与され、意思決定に利用できる形に成熟したものです。しかし、両者は対立する概念ではありません。データは情報の土台であり、情報が生まれるための原材料です。さらに情報は、適切な文脈のもとで活用されることで、知識や行動へとつながります。

重要なのは、どの段階で文脈と意味を付与するかを意識し、目的に応じて表現方法を選ぶことです。意味づけされていないデータは、ただ保存領域を圧迫するだけの存在になってしまいかねません。適切な文脈と形式を与え、データを情報へと成熟させることで、初めて価値ある知識につながります。

データが増え続ける現代において、情報として活用できる形に整えることは、無意味な蓄積を避け、判断や行動へとつなげるために欠かせない取り組みです。データを情報へと変える視点を持つことこそが、情報過多の時代を賢く生きるための鍵となるのです。
