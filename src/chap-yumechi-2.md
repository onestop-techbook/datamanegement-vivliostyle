# 自分の記憶管理のためのブログ執筆

最近、いろんなことを忘れつつある yumechi です。


私は寝ても覚めてもエンジニアリングの話をよくしています。
昨今のソフトウェアエンジニア領域ではAIの発展に伴い、多くの知識や用語を使って説明しなければならないことがあります。
しかし覚えられることには限界があります。さらに直近のAIの話ではなく、数年前のことであれば、さらに思い出すことも難しいでしょう。


そうした問題への解決方法としてブログの執筆を私は提案します。

## なぜブログなのか？

ブログ以外にも自分の考えをまとめる方法はいくつかあります。
手元でメモ帳に残すでも、Notion[^notion]やObsidian[^obsidian]といったツールでも管理できます。


しかし私はあえてブログという手段をとることが多いです。
その理由としては下記です。

- 自分自身の言葉でまとめやすいこと
- インターネットがあればどこからでも探しやすいこと
    - 自分の端末でなくても検索してその場で参照できる・してもらえること
- 検索機能が備わっていること、また大手検索エンジンを使っての検索が可能であること
    - 大手検索エンジンの場合は bot アクセスとのバランスの兼ね合いもあります
- 外部からのコメント・反応がもらえること

手元や自分だけがアクセス可能なサービスよりも人づてに探してもらえるブログは、強いメリットを持っている良い記録手段だと私は考えています。


具体的なエピソードとして、以前参加した勉強会で紹介されている新技術の話がありました。
私は技術名称を忘れていましたが、この時期のこの勉強会で聞いた気がするな、と思ってブログを検索したところ、無事にその技術の話を見つけることができました。


また旅行に行くことや遠出することもあるのですが、写真とともに感想や気持ちをブログに残すことで、いつでもどこでも思い出すことができます。

## 私がブログ機能に求めること

私がブログ機能に下記のことを求めています。

- 記事をたくさん書けること
- 記事の検索性が良いこと
- Markdown 記法で書けること
    - エンジニアならではの理由ではありますが、生産性の観点では重要です
- 記事の検索性が良い
    - サイトの信頼性が高い、SEOが高いことも重要です

上記のことを念頭に置き、私ははてなブログ[^hatenablog]を愛用しています。
はてなブログは上記の機能を有しており、アプリもあるので移動手段でも下書きに書き溜めておいて後からPCで清書できます。


Markdown 記法で書けるという条件を外れますが、note[^note]も良い選択肢です。
noteはエディタの機能が豊富で、Markdown記法をはじめとしたエンジニア向けのマークアップ言語を使わずとも書けるので、より初心者向きです。
利用可能な画像も多く、様々な表現が可能という点でも良いでしょう。


ほかの選択肢として、SEOの問題は気になりますが GitHub Pages[^githubpages] を使ってブログを作成するのも良いでしょう。
テンプレートとなるリポジトリやツールは無数にあるので、自分に合ったものを探してみるのもよいかもしれませんね。


## 仕事に関わることをブログに書く場合の注意点

個人的なことや仕事に関わることを書く際は、公開してはならない情報を避けなければなりません。
特にビジネスに関わることで外向けに公開されていない情報については、後ほど問題になる可能性が高いので避けましょう。
どうしてもという場合は会社に相談するのも一つの手でしょう。


仕事に関連したブログを書く場合は誰かにレビューしてもらうのも手でしょう。
気が付かないうちに使っていた社内の用語や言葉遣い、価値観が知らぬ間に出てしまうこともあるので、公開前になるべく見てもらいましょう。
何かあった時も一人で抱え込まず、心理的な負担を分散しておくことも大事です。


逆に個人的なことで思い出したいことはブログにどんどん書いておくのが良いでしょう。
旅行で行った場所とか、お店とか、その時の気持ちとかは後で思い出すのが大変です。
そういった素直な気持ちも含めて自分のポートフォリオとしてブログを作り上げていくのもまた面白いかもしれませんね。

## 今日から自分の記録をつけよう

ブログのネタを探すのは難しいですが、最初は思ったことを羅列してみるだけでも良いかと思います。
読んだニュースとか、興味のあるSNSのポストとかをメモしておいてやってみたい！とか、ここのお店行ったおいしかった！でも良いのでまずは書いてみるのが大事です。
素直に自分の気持ちを整理して表現してみましょう。


ただし下向きな話や他人を下げるような話は後悔しないほうがよいでしょう。
対人関係やその他のトラブルに繋がりかねないので、気持ちのコントロールも大事です。
文章に吐き出すと気持ちはすっきりするので、その時はプライベートな日記帳などに書くなど人目につかない場所に書いておくのも一つの手でしょう。


というわけでこの章を読んだ方のブログを楽しみにして、締めさせていただきます。

[^notion]: Notion: オールインワンのワークスペースツール。ドキュメント作成、データベース、タスク管理、Wiki などの機能が統合されており、チームでの共同作業にも適している。 https://www.notion.so/
[^obsidian]: Obsidian: ローカルファイルベースの Markdown 知識管理ツール。ノート間のリンク機能が強力で、知識をグラフ構造で可視化できる。プライバシー重視でオフライン利用が可能。 https://obsidian.md/
[^hatenablog]: はてなブログ: 日本の老舗ブログサービス。Markdown記法に対応し、技術系ブロガーに人気。無料プランでも多機能。 https://hatenablog.com/
[^note]: note: クリエイター向けのコンテンツ配信プラットフォーム。直感的なエディタで初心者でも使いやすく、有料記事の販売も可能。多様なジャンルのユーザー層を持つ。 https://note.com/
[^githubpages]: GitHub Pages: GitHub リポジトリを使って無料で公開できる静的サイトサービス。Markdown 記法で書けるので、エンジニアにとっては扱いやすい。 https://docs.github.com/ja/pages
