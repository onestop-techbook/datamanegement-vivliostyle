# 自分の記憶管理のためのブログ執筆

最近、いろんなことを忘れつつある yumechi です。

私の場合は寝ても覚めてもエンジニアリングの話をよくしているのですが、昨今の特にソフトウェアエンジニア領域ではAIの発展に伴い、多くの知識や用語を使う必要があります。
しかし覚えられることには限界がありますし、数年前のことであれば尚更思い出すことも難しいでしょう。

そうした問題への解決方法としてブログの執筆をおすすめします。

## なぜブログなのか？

ブログ以外にも自分の考えをまとめる方法はいくつかあります。
手元でメモ帳に残すでも、Notion[^notion]やObsidian[^obsidian]といったツールでも良いでしょう。

しかし私はあえてブログという手段をとることが多いです。理由としては下記です。

- 自分自身の言葉でまとめられていること
- どこからでも探しやすいこと
    - 自分の端末でなくてもインターネットがあれば探せる
- 検索機能が備わっていること、また大手検索エンジンを使っての検索が可能であること
    - 大手検索エンジンの場合は bot アクセスとのバランスの兼ね合いもあります
- 外部からの目から反応がもらえること

こう考えると、手元や自分だけがアクセス可能なサービスよりも人づてに探してもらえるなどのメリットもあり、ブログというのは良い手段だと考えています。

## ブログ機能に求めること

さて、ブログ機能に求めることとしては、記事をたくさん書けることと、記事の検索性が良いことです。
またエンジニアならば、 Markdown 記法で書けることも重要でしょう。
記事の検索性という観点では、そのブログサイト自体のサイト信頼性やSEOも大事になりそうです。

上記のことを念頭に置き、私ははてなブログ[^hatenablog]を愛用しています。
設定も容易で Markdown も標準的、かつ拡張して使えるサイト表示のタグなどもあるので重宝しています。無料でもかなりの機能が使えるのも強みです。

逆に Markdown 記法のような特殊な書き方を避けるのであれば、 note[^note] も選択肢となりそうです。 note も多機能な表現を備えつつも、ユーザー個別に記事を探す機能があります。

こういった観点で利用するサイトを考えてみるというのもまた面白いかもしれませんね。

## 個人的なことや仕事に関わること

個人的なことや仕事に関わることを書く際は、公開してはならない情報を避けましょう。特に外向けに公開されていない情報については、後ほど問題になる可能性が高いので避けましょう。どうしてもという場合は会社に相談するのも一つの手でしょう。

逆に個人的なことで思い出したいことはブログにどんどん書いておくのが良いでしょう。旅行で行った場所とか、お店とか、その時の気持ちとかはなかなか後で思い出すのが大変です。
そういった素直な気持ちも含めて自分のポートフォリオとしてブログを作り上げていくのもまた面白いかもしれませんね。

## 今日から自分の記録をつけよう

ブログのネタを探すのは難しいですが、最初は思ったことを羅列してみるだけでも良いかもと思います。
読んだニュースとか、興味のあるSNSのポストとかをメモしておいてやってみたい！とか、ここのお店行ったおいしかった！でも良いのでまずは書いてみるのが大事です。

というわけでこの章を読んだ方のブログを楽しみにして、この章の締めとさせていただきます。

[^notion]: Notion: オールインワンのワークスペースツール。ドキュメント作成、データベース、タスク管理、Wiki などの機能が統合されており、チームでの共同作業にも適している。 https://www.notion.so/
[^obsidian]: Obsidian: ローカルファイルベースの Markdown 知識管理ツール。ノート間のリンク機能が強力で、知識をグラフ構造で可視化できる。プライバシー重視でオフライン利用が可能。 https://obsidian.md/
[^hatenablog]: はてなブログ: 日本の老舗ブログサービス。Markdown記法に対応し、技術系ブロガーに人気。無料プランでも多機能。 https://hatenablog.com/
[^note]: note: クリエイター向けのコンテンツ配信プラットフォーム。直感的なエディタで初心者でも使いやすく、有料記事の販売も可能。多様なジャンルのユーザー層を持つ。 https://note.com/
